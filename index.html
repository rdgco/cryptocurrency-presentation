Cryptocurrency

  • Uses a blockchain and cryptography for security.
  • Not issued by any central authority.
  • The amount of coins available is determined by transactions in the blockchain.
  • Network consensus is used to improve the network technology over time.
  • Coins are introduced to the blockchain via mining rewards.
Who invented bitcoin?
Satoshi Nakamoto
  • Wrote the white paper "Bitcoin: A Peer-to-Peer Electronic Cash System" in 2009.
  • https://bitcoin.org/bitcoin.pdf
  • Invented blockchain technology for cryptocurrencies.
  • Estimated owned bitcoins: 980,000 bitcoins.
  • Worth almost 10 billion dollars in current market.
  • Nobody really knows who this person or group of people are.
  • Has since handed off control of bitcoin to other developers.

Blockchain Technology

A blockchain is a distributed ledger of transactions that is publicly verifiable and historically unchangeable.
Ledger
  • A ledger is a record of transactions.
  • Transactions on a blockchain describe the change of state in the ledger.
  • Groups of transactions are combined together by time into blocks.
  • The amount of time for each block generation varies by network.
  • Bitcoin blocks are generated every ten minutes
Distributed
  • A blockchain is a publicly available set of data.
  • A peer to peer network of nodes maintains the blockchain state.
  • Each node has a full copy of the blockchain.
  • The process of creating new blocks is called mining.
How large are blockchains?
  • Bitcoin Now: 144 Gig.
  • Bitcoin Upper Limit: 3TB.
  • Full mining nodes must download a copy of the whole blockchain.
  • The size of the blockchain is a challenge for scalability.

Some underlying technologies

  • Public Key Cryptography
  • Secure Hash Based Identifiers
  • Peer to Peer Networking
  • Proof of Work

Public Key Cryptography

  • An algorithm generates a public key and private key pair.
  • The private key must be kept secret by the owner.
  • The private and public keys are cryptographically related.
What is public key cryptography used for?
  • The holder of the private key can digitally sign a message.
  • The message signature can be verified with the public key.
  • 2 parties can communicate with each other and be assured that the sender holds the private key.
  • Control and security of the private key is critical.
Public Key Cryptography and the Bitcoin Blockchain
  • Blockchain addresses are based on the public key.
  • The holder of a private key can issue transactions from the associated Blockchain address.
  • A transaction can be issued to a bitcoin address with the sender's private key and the receiver's bitcoin address.

Encryption and Cryptography

  • A third party can encrypt a message with the receiver's public key.
  • The holder of the private key can decrypt messages that were encrypted with the private key.
  • Combined with signing, two parties can communicate with each other confidently and securely without any eavesdropping unless a third party has the privarte key

Secure Hash Based Identifiers

  • Input: Any data of any length.
  • Output: A Fixed size alphanumeric string.
  • Example Output: 2b12fcf1b09288fcaff797d71e
  • Calculating a hash value for data is computationally easy.
  • It is computationally difficult to generate original text from a hash.
  • It is very unlikely that two different inputs yield the same hash.

Example Uses of Hash Functions

  • Password authentication
  • The best practice method for authenticating is to never save the password on the server. Only store a hash of the password.
  • Source Code Versioning
  • In git, a cryptographic hash is used to identify the entire state of all files within a version of blockchain node software
  • Blockchains
  • Hash functions are used to uniquely identify transactions, blocks, and the entire blockchain sequence.
  • Each block has a final hash key that is dependent upon all transactions in that block, as well as all blocks that have been mined previously
Peer to Peer Networks
  • A network of nodes which follow a protocol to communicate with each other.
  • Unlike client/server with two tiers.
  • Bittorrent is a peer to peer network for sharing files.
  • Blockchain mining is done on peer to peer networks.
Proof of Work
  • A mathematical problem that is difficult to solve.
  • The solution to the problem is easy to validate.
  • A node proves that it performed work by solving the problem.
  • The network reaches consensus as other nodes validate the solution.
Bitcoin Proof of work
  • A mining node generates a hash code for the block of transactions.
  • The hash code must start with a number of leading zeros.
  • The node can generate different hash codes by changing one value in the transaction called the Nonce.
  • The node will continue to make attempts until it generates a matching hash.
  • The first node to find the matching hash wins the mining award.
Increasing the difficulty in Bitcoin Over Time
  • Periodically, the number of required leading zeros is increased.
Proof of work uses a lot of energy globally as each node tries to mine each subsequent block.
Proof of Stake is an alternative mechanism that determines the miner for each block based on the percentage of currency assigned to an account for the miner. This saves energy by limiting the number of nodes that will attempt to mine the next block, and the mathematical problem no longer needs to be difficult to solve. It could possibly be as simple as creating the correct hash.

Distributed Open Source Software Development

  • Open source software is developed by individuals and companies which collaborate globally.
  • Open source licenses allow others to use and modify code without paying money.
  • Sites such as github.com are the 'coding networking' hub for open source development.
  • The software for running each type of node is developed in a collaborative environment.
  • Nodes for the same blockchain can and are implemented in different languages

Ethereum Blockchain

  • Invented by Vitalik Buterin 2013.
  • Age 23 today.
  • Ethereum implements smart contracts in the blockchain.
  • A smart contract is a computer program that runs on a virtual machine in the blockchain.
  • A virtual machine is a computer that is implemented in software by each node.
  • The blockchain contains both the computer program, and the state that it generates when transactions are executed.
  • Public functions can be invoked in a smart contract by other smart contracts.
Initial Coin Offering (ICO)
  • Initial Coin Offerings are simply smart contracts that keep track of how many coins there are and what accounts they belong to.
  • Security holes and careless programming in smart contracts has led to losses of close to a hundred million dollars of virtual currency.
  • The SEC has declared that ICO's are illegal in the United States unless they are licensed securities.
Distributed Apps - Dapps
  • An integration between the web browser, smart contracts, and blockchains via peer to peer protocols.
  • Ethereum allows contracts to be invoked from the web browser.
  • CryptoKitties is now the most famous Dapp.
  • The contract allows people to buy and breed digital cats.
  • Its popularity has slowed transaction processing on Ethereum.
